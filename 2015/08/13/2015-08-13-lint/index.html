Lint的使用与配置

使用

Android studio默认已经集成了Lint，使用起来非常简单：

Analyze -> Inspect Code

然后选择要检查的文件，或者整个project，或者某个module，或者某个文件。然后执行后，会在左下角出现检测到的问题列表，点击某个issue后，在右侧会出现具体的描述，以及潜在的解决方案。

所以建议经常做lint检测，对我们的写代码能力的提高也很有帮助。

配置

在做lint检测的时候，检查哪些项目？某个具体的项目检测出来具体以warning的形式还是以error的形式呈现出来？这一系列的选择都是可以配置的：

File -> Other settings -> Default Settings -> Editor -> Code Style -> Inspections

另外，可以在build.gradle里面做一些lint执行的相关配置，例如：

android {
lintOptions {
   // set to true to turn off analysis progress reporting by lint
   quiet true
   // if true, stop the gradle build if errors are found
   abortOnError false
   // if true, only report errors
   ignoreWarnings true
   }
   ...
}

lint.xml

当需要排除某些文件的lint的检测时，也就是为某些文件的某些lint问题申请赦免的时候，这个时候lint.xml就派上用场了，看看下面这个例子：

<?xml version="1.0" encoding="UTF-8"?>
<lint>
    <!-- Disable the given check in this project -->
    <issue id="IconMissingDensityFolder" severity="ignore" />

    <!-- Ignore the ObsoleteLayoutParam issue in the specified files -->
    <issue id="ObsoleteLayoutParam">
        <ignore path="res/layout/activation.xml" />
        <ignore path="res/layout-xlarge/activation.xml" />
        </issue>

    <!-- Ignore the UselessLeaf issue in the specified file -->
    <issue id="UselessLeaf">
        <ignore path="res/layout/main.xml" />
    </issue>

    <!-- Change the severity of hardcoded strings to "error" -->
    <issue id="HardcodedText" severity="error" />
</lint>
