Android Studio Live Template

Posted on 2015-03-09   |   In Android   |  

代码的复用在开发过程中为我们省却不少重复性的工作，从而提高开发效率。目前大部分的IDE都支持了code snippet，Android studio也不例外，今天我们就一起来看看Android studio中Live template(Android studio将code snippet称为live template)的创建与使用。

使用已有的Live Template

Android studio默认已经集成了一些live template进去，也就是我们在开发过程中可以快速的插入一些代码块。

下面是个简单的例子：

假如我们的代码中定义了一个list:

List<String> list = Arrays.asList("a", "b", "c");

现在想遍历这个list，我们只需要敲入：

itli

然后按Tab键，这时Android studio就会自动插入遍历list的代码：

List<String> list = Arrays.asList("a", "b", "c")

for (int i = 0; i < list.size(); i++) {
    Object s =  list.get(i);

}

怎么样，是不是很方便？

创建自定义的Live Template

随着开发的深入，我们会发现：仅仅使用Android studio自带的live template不能满足我们的需求的。有时候我们想把自己的一些代码存成live template，以便以后能够方便的复用。 其实方法也很简单：

首先选中要创建的代码，然后选择tools-> save as Live Template，命名保存即可。

接下来也举个例子：
比如开发中常见的单例模式，我们有个类名字为ClassA，然后想使其成为单例模式，其中createInstance函数可以这样写

private static ClassA mInstance = null;
public static ClassA createInstance() {
    if(mInstance == null) {
        mInstance = new $variable$();
    }

    return mInstance;
}

然后选中这段代码，点击菜单Tools -> save as Live Template, 命名为singleton保存。等以后使用时直接敲入singleton然后按tab键，那么上面的一大段代码就自动插入进来了。

然后问题总是不期而至，使用了一段时间之后，你又发现了问题：不管我的类名是什么，插入的代码总是ClassA！！ 怎么处理呢？ 还是有办法：
我们把template中的ClassA改成一个变量的形式，例如$variable$, 改完后如下：

private static $variable$ mInstance = null;
public static $variable$ createInstance() {
    if(mInstance == null) {
        mInstance = new $variable$();
    }

    return mInstance;
}

小结

通过以上介绍，详细你已经了解了Android Studio的代码模板的使用以及创建。 这样在以后的开发中可以尽量少的重复创建轮子。 不过既然想拒绝重复创造，我们其实还可以直接引入别人已经创建好的template，google一下，你会发现已经有一些不错的template可以直接拿来用了。
